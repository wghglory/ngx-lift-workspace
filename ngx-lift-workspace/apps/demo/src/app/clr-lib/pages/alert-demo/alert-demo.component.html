Alerts are banners that draw the user's attention to important messages. Visual elements such as icons and colors indicate the type and urgency of the information being communicated.

Usage

Use alerts to provide critical information that requires immediate user attention within the current context.

The <cll-alert /> component accepts the following optional inputs:

  • alertType: One of "info", "warning", "success", "danger", "neutral", "loading", or "unknown". Defaults to "danger".
  • isAppLevel: Boolean flag for app-level alerts. Defaults to false. Set to true to display an app-level alert.
  • isSmall: Boolean flag for compact sizing. Defaults to false.
  • isLightweight: Boolean flag for lightweight styling. Defaults to false. Set to true to display a lightweight alert.
  • error: An HttpErrorResponse object to display error information.
  • content: A string containing the alert message content.

For complex HTML content, the component provides an ng-content projection slot for custom content injection.

Types

There are 3 types of alerts: Standard, Lightweight and App-level.

Standard Alerts

Use standard alerts for messages that reflect the context of where the user is in the application.

Display an API error response

When displaying error responses from APIs, use the error input to pass error details. The alert displays the error message from the message property of the response error object if available. Otherwise, it falls back to displaying the message from the HttpErrorResponse object.

Display a custom content

The content is rendered using innerHTML, allowing you to include HTML tags. This is particularly useful when working with translation strings that contain HTML markup, enabling rich text formatting and structured content in alerts.

Inject HTML tags by content projection

The alert component also supports content projection via ng-content, enabling the inclusion of complex HTML structures and Angular components.

Lightweight Alerts

Use lightweight alerts inside containers, other components and layouts.

App-Level Alerts

Use app-level alerts in the global context of an application. Place them at the top of all content and navigation.
